# 数据库设计

## 1. 设计约定

数据库推荐 PostgreSQL 16。所有业务表建议包含以下通用字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | uuid/bigserial | 主键，推荐 uuid |
| created_at | timestamptz | 创建时间 |
| updated_at | timestamptz | 更新时间 |
| created_by | uuid | 创建人 |
| updated_by | uuid | 更新人 |
| deleted_at | timestamptz | 软删除时间，可选 |

命名规范：

- 表名使用复数或业务名，统一 snake_case。
- 枚举状态使用 varchar，应用层维护枚举。
- 大字段、动态字段使用 jsonb。
- 高频查询字段建立索引。
- 涉及项目隔离的表必须包含 project_id。

## 2. ER 关系概览

```text
users -- user_roles -- roles -- role_permissions -- permissions
users -- teams -- team_members
users -- invitations
users/teams -- project_members -- projects -- project_roles -- project_role_permissions
projects -- requirements -- requirement_versions
projects -- test_cases -- test_case_versions
requirements -- test_case_requirements -- test_cases
projects -- review_requests -- review_comments
projects -- test_plans -- test_suites -- test_suite_cases
projects -- execution_runs -- execution_case_results -- execution_step_results
projects -- devices -- device_locks
projects -- integrations
projects -- configuration_items -- configuration_versions
projects -- tool_definitions -- tool_instances -- tool_invocations
projects -- assistant_sessions -- assistant_messages -- assistant_actions
projects -- reports
projects -- notifications
ai_tasks -> requirements/test_cases/reports/assistant_actions
```

## 15. 分区与归档建议

执行结果和日志增长快，建议：

- `execution_runs` 按创建时间或项目归档。
- `execution_case_results` 按月分区。
- `execution_artifacts` 文件存储在 MinIO，数据库只保存索引。
- `audit_logs` 至少保留 180 天，可归档到冷存储。
- `ai_tasks` 保留输入摘要，不建议长期保存完整敏感需求原文，除非项目允许。
