# AutoTestHub 完整测试流程指南

## 🚀 快速开始

### 1. 启动后端服务

```bash
cd /Users/bytedance/Documents/AutoTestHub/backend
source venv/bin/activate
python -m app.seed  # 初始化数据库（如果还没运行过）
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 2. 启动前端服务

```bash
cd /Users/bytedance/Documents/AutoTestHub/frontend
npm install  # 首次运行需要安装依赖
npm run dev
```

### 3. 启动 Agent（可选，用于实际执行测试）

```bash
cd /Users/bytedance/Documents/AutoTestRunner
source venv/bin/activate
pip install psutil requests  # 首次运行需要安装 Agent 依赖
python -m autotest_runner.agent.agent --server http://localhost:8000 --name "本地测试Agent"
```

### 4. 访问系统

打开浏览器访问：`http://localhost:5173`（或前端显示的地址）

默认账号：
- 用户名：`admin`
- 密码：`AutoTestHub@123`

## 📋 完整测试流程演示

### 第一步：创建项目

1. 登录系统后，点击首页的「新建项目」
2. 填写项目信息：
   - 名称：`演示项目`
   - 描述：`用于演示完整测试流程`
3. 点击「创建」

### 第二步：创建需求和测试用例

**方式一：手动创建用例**

1. 进入项目详情页 →「需求管理」→「新建需求」
2. 填写需求：
   - 标题：`用户登录功能`
   - 描述：`用户输入用户名和密码进行登录，成功后跳转到首页`
3. 点击「创建」
4. 点击「AI生成用例」按钮（可选，需要配置 AI 服务）
5. 或者手动创建用例：
   - 进入「测试用例」→ 点击「用例管理」→「新建用例」

**方式二：本地导入用例（高级）**

AutoTestRunner 提供了本地用例同步功能。

### 第三步：启用测试用例

1. 进入「测试用例」标签页
2. 选择要执行的用例
3. 将状态从「候选」→「采纳」→「启用」

### 第四步：立即执行测试

1. 在「测试用例」或「执行记录」标签页，点击「立即执行」
2. 填写执行信息：
   - 名称：`演示测试执行`
   - Agent：选择可用的 Agent（如果有）
   - 优先级：默认
3. 点击「确定」
4. 系统会创建执行记录并加入队列

### 第五步：创建定时任务（可选）

1. 进入「定时任务」标签页
2. 点击「新建定时任务」
3. 填写任务信息：
   - 名称：`每日回归测试`
   - Cron表达式：选择「每天早上8点」或输入：`0 8 * * *`
   - 启用：✓
4. 点击「创建」
5. 可以点击「触发」立即测试一次

## ⏰ 定时任务配置说明

### Cron 表达式格式

```
┌───────────── 分钟 (0 - 59)
│ ┌───────────── 小时 (0 - 23)
│ │ ┌───────────── 日期 (1 - 31)
│ │ │ ┌───────────── 月份 (1 - 12)
│ │ │ │ ┌───────────── 星期 (0 - 6，周日是0)
│ │ │ │ │
* * * * *
```

### 常用示例

| 表达式 | 说明 |
|--------|------|
| `* * * * *` | 每分钟执行一次 |
| `*/5 * * * *` | 每5分钟执行一次 |
| `0 * * * *` | 每小时执行一次 |
| `0 8 * * *` | 每天早上8点执行 |
| `0 20 * * *` | 每天晚上8点执行 |
| `0 8 * * 1` | 每周一早上8点执行 |
| `0 8 1 * *` | 每月1号早上8点执行 |

## 🔧 Agent 管理

### Agent 状态说明

- **在线（绿色）**：Agent 正常连接，可以接收任务
- **忙碌（黄色）**：Agent 正在执行任务
- **离线（灰色）**：Agent 断开连接

### Agent 资源监控

在「Agent管理」页面可以查看：
- CPU 使用率
- 内存使用情况
- 连接的设备列表
- 执行任务历史

## 📊 执行流程说明

```
1. 创建测试执行请求
   ↓
2. 系统创建 ExecutionRun 记录
   ↓
3. 为每个用例创建 ExecutionResult 记录（状态：待执行）
   ↓
4. 任务加入 ExecutionQueue 队列
   ↓
5. Agent 轮询获取任务
   ↓
6. Agent 执行测试（实际测试逻辑）
   ↓
7. Agent 更新 ExecutionResult 状态
   ↓
8. 更新 ExecutionRun 状态为完成
   ↓
9. 可以查看报告和 AI 分析
```

## 🤖 AI 功能使用（可选）

### 1. 配置 AI 服务

1. 进入「系统设置」→「模型配置」
2. 选择 AI 提供商（如 OpenAI、Claude、通义千问等）
3. 填写 API Key 和其他配置
4. 点击「测试连接」→「保存」

### 2. AI 生成用例

1. 在需求详情页，点击「AI生成用例」
2. 系统会自动生成测试用例
3. 查看并采纳合适的用例

### 3. AI 分析报告

1. 进入执行报告详情页
2. 点击「AI分析报告」
3. 查看 AI 生成的测试分析和建议

## 💡 最佳实践建议

### 1. 项目管理

- 按产品线或测试阶段创建独立项目
- 合理使用需求来组织功能模块
- 定期清理不再需要的旧执行记录

### 2. 测试用例管理

- 启用状态的用例才会被包含在执行中
- 合理设置优先级（P0最高）
- 用 AI 生成用例后，需要人工审核和补充
- 使用「特性」字段来分类管理用例

### 3. 定时任务

- 先手动测试确认流程正常，再设置定时任务
- 定时任务建议在低峰期执行（如深夜）
- 为定时任务设置合适的超时时间
- 监控执行失败的任务并及时处理

### 4. Agent 部署

- Agent 应该部署在靠近测试设备的机器上
- 为不同的测试环境配置不同的 Agent
- 定期检查 Agent 状态和资源使用情况

## 🔍 故障排查

### Agent 连接失败

1. 检查后端服务是否正常运行
2. 确认 Agent 配置的服务器地址正确
3. 检查网络连接和防火墙设置
4. 查看 Agent 日志获取详细信息

### 测试任务一直待执行

1. 确认有在线的 Agent
2. 检查 Agent 状态是否正常（在线且不忙）
3. 查看 Agent 日志看是否有报错

### 数据库问题

1. 运行 `python -m app.seed` 重新初始化数据库
2. 检查数据库文件权限
3. 查看后端日志
